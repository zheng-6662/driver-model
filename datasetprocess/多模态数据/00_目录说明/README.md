# 多模态数据目录说明

这个目录现在按“核心数据/核心代码/运行产物/归档资料”来维护，尽量减少顶层散乱文件。

## 现在顶层主要保留什么

- `被试数据集合/`
  - 多模态主数据源，按被试组织的 vehicle / physio / eeg / event。
- `Event_Dataset/`
  - 早期事件级数据集。
- `Event_Dataset_v2/`
  - 当前更值得优先使用的事件级数据集版本。
- `数据处理代码/`
  - 多模态数据处理相关旧代码和分模块脚本。
- `模型代码/`
  - 当前模型主代码目录，包含最新训练脚本和诊断脚本。
- `程序运行结果/`
  - 训练与评估的运行产物。
- `corr_outputs_v5_1_FULLSET/`
  - 相关性分析输出。
- `eeg_feature_outputs_v312/`
  - EEG 特征输出。
- `eeg_qc_clean_fif_v3/`
  - EEG 清洗后的 fif 结果。
- `eeg_qc_outputs/`, `eeg_qc_outputs_v2/`, `eeg_qc_outputs_v3/`
  - EEG 质控输出。

## 维护规则

1. 新的 `.pth` 权重不要再直接放顶层，放到 `02_模型权重归档/` 或单独运行目录。
2. 新的结果图、散点图、预测图、结果表不要再直接放顶层，放到 `03_分析结果归档/` 或具体运行目录。
3. 临时表格、压缩包、说明文本不要再放顶层，统一放 `04_资料归档/`。
4. 代码优先收敛到 `final_code/` 或 `模型代码/`，不要继续在顶层新增临时脚本。
